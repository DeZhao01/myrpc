# MyRPC - 一个轻量级 RPC 框架（C++17）

## 项目简介
这是一个从零实现的 RPC（远程过程调用）框架，旨在学习 C++ 网络编程、协议设计和高性能并发模型。项目实现了自定义二进制协议，基于 Linux epoll 的边缘触发模式构建单线程 Reactor 网络层，支持异步消息处理，并提供了序列化/反序列化接口。

## 技术栈
- **语言**：C++17
- **网络**：Linux socket + epoll（边缘触发，非阻塞 I/O）
- **协议**：自定义二进制协议（固定头部 + 可变长度消息体）
- **构建**：CMake
- **并发**：单线程事件循环（可扩展为线程池）

## 核心特性
- 自定义 RPC 协议（魔数校验、版本控制、消息类型、消息 ID、长度字段）
- 高效的网络层：epoll 边缘触发 + 非阻塞 I/O，避免线程阻塞
- 每个连接独立缓冲区，处理 TCP 粘包问题
- 回调解耦：网络层与业务逻辑分离，易于扩展
- 完整的序列化/反序列化实现（支持二进制安全）
- 包含测试客户端，演示请求-响应流程

## 快速开始

### 环境要求
- Linux 系统（或 WSL2）
- CMake 3.16+
- 支持 C++17 的编译器（g++ 7+ / clang 6+）

### 构建
```bash
git clone <your-repo-url>
cd myrpc
mkdir build && cd build
cmake .. && make
```

### 运行
1. **启动服务器**：
   ```bash
   ./myrpc
   ```
   服务器默认监听 8888 端口。

2. **运行客户端**（新终端）：
   ```bash
   ./client
   ```

3. **预期输出**：
    - 服务器端会打印接收到的请求和发送的响应。
    - 客户端会显示连接、发送、接收和解析结果。

## 核心设计要点
- **Pimpl 模式**：隐藏网络层实现细节，降低编译依赖。
- **边缘触发**：监听套接字和客户端套接字均使用 `EPOLLET`，要求事件循环中一次性处理所有就绪事件。
- **缓冲区管理**：每个连接拥有独立 `std::vector<char>` 缓冲区，预分配 4KB 以减少动态扩容。
- **协议安全**：魔数校验 + 长度检查，防止解析非法数据。

## 后续可优化方向
- 添加线程池，将业务处理与 I/O 线程分离。
- 实现服务注册与发现（如基于 `std::unordered_map` 映射方法名到函数）。
- 完善粘包处理：在回调中循环解析缓冲区，直到无法解析出完整消息。
- 支持异步发送（处理 `EAGAIN`）和超时重传。
- 添加更丰富的协议字段（如校验和、扩展头部）。
